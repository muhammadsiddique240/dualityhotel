# DualityHotel - Deployment Guide

## Option 1: Deploy on Vercel (Recommended - Free)

### Prerequisites
- Vercel Account (free at vercel.com)
- GitHub Account with your code pushed

### Step-by-Step Deployment

1. **Push Your Code to GitHub:**
   ```bash
   git add .
   git commit -m "Prepare for Vercel deployment"
   git push origin master
   ```

2. **Go to Vercel Dashboard:**
   - Visit https://vercel.com/dashboard
   - Click "New Project"
   - Select your GitHub repository "dualityhotel"

3. **Configure Project:**
   - Framework: "Other"
   - Root Directory: "./"
   - Build Command: `pip install -r requirements.txt && python manage.py collectstatic --noinput && python manage.py migrate`
   - Install Command: `pip install -r requirements.txt`
   - Start Command: `gunicorn dualityhotel.wsgi`

4. **Add Environment Variables:**
   - Click "Environment Variables"
   - Add these variables:
     ```
     DJANGO_SECRET_KEY=<generate-a-random-secret-key>
     DEBUG=False
     ALLOWED_HOSTS=<your-vercel-domain>.vercel.app
     ```

5. **Deploy:**
   - Click "Deploy"
   - Wait for deployment to complete
   - Your site will be live at: `https://<project-name>.vercel.app`

## Option 2: Deploy on Railway.app (Better for Django)

### Prerequisites
- Railway Account (free tier available at railway.app)
- GitHub Account

### Step-by-Step Deployment

1. **Push Your Code to GitHub:**
   ```bash
   git add .
   git commit -m "Prepare for Railway deployment"
   git push origin master
   ```

2. **Go to Railway Dashboard:**
   - Visit https://railway.app
   - Click "New Project"
   - Select "Deploy from GitHub repo"
   - Choose your dualityhotel repository

3. **Configure Environment Variables:**
   - Go to Variables tab
   - Add these:
     ```
     DJANGO_SECRET_KEY=<random-secret-key>
     DEBUG=False
     ALLOWED_HOSTS=*.railway.app
     DATABASE_URL=<auto-generated-if-using-postgres>
     ```

4. **Deploy:**
   - Railway will auto-detect Django and deploy
   - Your app will be live in 2-5 minutes

## Option 4: Deploy on Heroku (Paid)

1. Create Heroku account at heroku.com
2. Install Heroku CLI
3. Create Procfile:
   ```
   web: gunicorn dualityhotel.wsgi
   ```
4. Deploy:
   ```bash
   heroku login
   heroku create your-app-name
   git push heroku main
   ```

## Common Issues & Fixes

### Issue: 502 Bad Gateway
**Solution:** Check environment variables are set correctly and database migrations ran.

### Issue: Static Files Not Loading
**Solution:** Run `python manage.py collectstatic` during build.

### Issue: Database Connection Error
**Solution:** Use PostgreSQL instead of SQLite for production.

---

## Production Database Setup (PostgreSQL)

1. Create PostgreSQL database on:
   - Vercel Postgres (free)
   - Railway Postgres
   - AWS RDS
   - DigitalOcean

2. Update `settings.py` to use PostgreSQL:
   ```python
   DATABASES = {
       'default': {
           'ENGINE': 'django.db.backends.postgresql',
           'NAME': os.environ.get('DB_NAME'),
           'USER': os.environ.get('DB_USER'),
           'PASSWORD': os.environ.get('DB_PASSWORD'),
           'HOST': os.environ.get('DB_HOST'),
           'PORT': os.environ.get('DB_PORT', '5432'),
       }
   }
   ```

3. Add `psycopg2-binary` to requirements.txt
